一、使用hexo搭建个人博客：

（一）环境配置

1.安装node和npm（node包管理工具）：

brew install node

安装结束后通过node -v和npm -v命令查看node和npm的版本，成功出现版本号说明安装成功

2.全局安装hexo插件（hexo也是node的一个插件？）：

npm install -g hexo-cli

安装后在命令行输入hexo，如果没报错说明hexo安装成功

（二）初始版本

1.新建博客文件夹~/Documents/MyBlogSrc/，进到该目录后执行hexo init，完成初始化。命令执行后本地会生成如下的文件：此外还有隐藏的git文件，说明这已经是一个本地的git仓库了

发布新博客的步骤：编写博客文件->生成静态文件->本地查看生成的静态文件，检查是否有问题->将生成的静态文件部署到服务器上（这里采用git作为服务器）。

2.假设新增博客文件：myblog.md，文件放置在source/_posts/文件夹下，对应的图片可以选择上传至图床，也可以放在本地source/_imgs文件夹下，可以直接将md文件放置在对应路径，也可以执行hexo new “myblog”命令生成对应的博客源文件

3.新增后，执行hexo generate，生成静态文件。

4.执行hexo server，开启本地服务器，检查生成的静态部署文件是否正确

5.确认无误后，将生成的静态文件部署至远程服务器，通过执行hexo deploy命令实现。注意，执行该命令后，可以看到文件夹下新增了一些文件
