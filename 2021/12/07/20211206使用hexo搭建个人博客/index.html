使用hexo搭建个人博客

发表于 2021-12-07

这篇记录一下怎么使用hexo来搭建个人博客，以及过程中遇到的一些问题～

一、使用hexo搭建个人博客

（一）环境配置

1.安装node和npm（node包管理工具）：

brew install node

安装结束后通过node -v和npm -v命令查看node和npm的版本，成功出现版本号说明安装成功。

2.全局安装hexo插件：

npm install -g hexo-cli

安装后在命令行输入hexo，如果没报错说明hexo安装成功。

（二）初始版本

1.文档结构

新建博客文件夹~/Documents/MyBlogSrc/，进到该目录后执行hexo init，完成初始化。命令执行后本地会生成如下的文件：此外还有隐藏的git文件，说明这已经是一个本地的git仓库了

发布新博客的步骤：编写博客文件->生成静态文件->本地查看生成的静态文件，检查是否有问题->将生成的静态文件部署到服务器上（这里采用git作为服务器）。

2.新增博客

假设新增博客文件：myblog.md，文件放置在source/_posts/文件夹下，对应的图片可以选择上传至图床，也可以放在本地source/_imgs文件夹下，可以直接将md文件放置在对应路径，也可以执行hexo new "myblog"命令生成对应的博客源文件。然后就可以使用typora编辑博客了～

（1）图片路径设置

博客中的图片可以上传至免费图床，但感觉不是很方便，最后选择将图片保存在本地。如果写markdown文件时文件路径没有正确设置，可能会导致部署至远程服务器后图片不能正常显示。以下是解决方案：

  • 设置站点配置文件_config.yml的”post_asset_folder”字段的属性为”true”；

  • 对于某一篇博客文章xx.md（位于source/_posts/目录下），同时在该目录下新建对应的同名的图片资源文件夹，也就是xx.md对应一个名为xx的文件夹，这一步也可以直接使用hexo new xx实现（但是是在”post_asset_folder”字段的属性为”true”的前提下）；

  • 设置typora中的图片相关属性：typora偏好设置->图像->插入图片时->复制到指定路径，如下图所示，这样设置之后，当复制图像到t ypora中时就会将图片存储在md文件路径下的同名文件夹中，这样上一步新建博客文章时，也无需再手动新建同名文件夹了。

    复制到typora中的图片路径应该如下图所示（中间一长串是因为文件夹名字有中文）：

  • 安装插件hexo-asset-image（插件安装后都存在node_modules文件夹中），目的是在hexo将md文件转为html静态文件时，将md文件中的相对路径转为html文件中的绝对路径：npm install https://github.com/CodeFalling/hexo-asset-image --save，注意，如果直接使用npm install hexo-asset-image --save安装的插件会有问题，转换后的路径会多一个”.com//“；

3.本地预览

  • 博客文件编辑完成后，执行hexo clean和hexo generate命令，会在public文件夹下生成对应的静态网页文件，对应路径通过博客生成的日期指示。

  • 接着执行hexo server，开启本地服务器，检查生成的静态部署文件是否正确，也可以hexo server –debug进入调试模式。

4.部署到远程服务器

确认无误后，需要将生成的静态文件部署至远程服务器，这里选择将其部署在免费的GitHub Pages，主要包含以下步骤：

  • 执行npm install hexo-deployer-git –save命令，安装对应的工具，并将该包添加至package.json，便于后续迁移；

  • 在GitHub上新建一个名为”username.github.io”的仓库，其中username为GitHub的账户名；

（三）进阶版本

在了解基本的博客发布流程后，可以对博客进行一些优化，包括更换主题、调整样式等。

1. 更换主题

更换博客的默认主题为NexT主题，步骤如下：

  • 在GitHub上讲NexT主题的对应仓库fork到自己的账号下;

  • 进入博客仓库的根目录下，把主题对应的仓库添加为我们的博客的仓库的一个子模块（后续主题仓库有更新时可以pull下来）：

    git submodule add https://github.com/<username>/hexo-theme-next themes/next

    主题相关代码会放置在themes/next文件夹中，其中的_config.yml文件为主题配置文件，用于设置一些主题相关的属性；

  • 更改站点配置文件（即博客根目录下的_config.yml），修改”theme”字段的属性为”next”;

  • 执行hexo clean;hexogenerate;hexo server，在本地预览所设置的主题是否生效。

二、多台电脑共同管理博客

方式1:源代码和发布后生成的静态文件分别放在同一git项目的两个分支，缺点：所有源代码为公开的；

方式2:源代码和发布后生成的静态文件分别放在两个git项目中，源代码对应的git仓库设置为私有的。

采用第2种方式：

假设博客源代码放在路径：~/Documents/MyBlogSrc/，在项目根目录下初始化git仓库，在GitHub上新建一个私有仓库，将代码push到GitHub。如果需要在另外一台电脑上更新博客
